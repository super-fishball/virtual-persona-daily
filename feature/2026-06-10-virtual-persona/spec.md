# 总 spec —— F1 首刀 A1：创建虚拟人 · 单次生成 · 时间线

> feature：`2026-06-10-virtual-persona`｜本文件 = 跨子项目**总 spec + 设计理由（why）**。
> 契约「是什么」（路由 / 请求响应 / 错误码）**不在本文件**——只有一份权威源 = 后端机读 schema，由 **F2** 落地，指针进 [`split.md`](split.md)（本期尚未建）。本 spec 只描述**意图、方向、边界与 why**。
> 状态：AI（brainstorming）起草 → 待涉及方（4 子项目开发者）集体评审定稿。契约僵持时由提供方（后端）兜底。

## 1. 范围：首刀 A1（只做这些）

A1 是整套产品（见 [`requirement.md`](requirement.md)）的**第一刀**，目标是**用最小切片一刀穿透 4 子项目 + 3 条契约 + 3 个高影响区**，验证整条骨架与协作流程，而非交付完整产品。

A1 = 三件事：

| 编号 | 故事 | A1 内涵 |
|---|---|---|
| **U0** | 创建虚拟人 | 真实浏览器定位 → 高德逆地理编码取城市 → 落 `home` 坐标 + 1 个 `place`；用户指定性格 |
| **单次生成** | 生成一个事件 | 创建时**同步**经 ai-gateway 调 LLM 生成**首个"诞生"事件**（含 `end` 时间）；guardrails 三层就位 |
| **U1** | 时间线列表 | 看当天事件序列（A1 即这一条诞生事件起步的列表） |

**一刀穿透：**
- **4 子项目**：`apps/web` → `servers/api` → `servers/generation-service` → `servers/ai-gateway` → DeepSeek。
- **3 条契约**：web↔api（对外）、api↔gen（内部）、gen↔ai-gateway（内部）。**本 spec 只立边界与意图，schema 是 F2。**
- **3/5 高影响区**（[`high-impact.md`](../../rules/high-impact.md) 标 ⚡ 的三类，首刀即生效）：性格→prompt 注入面、高德 key 在 gen、定位 PII 在 web。

A1 **明确不做**的部分集中在 [§7](#7-首刀明确不做范围外)，spec 全文不依赖任何被排除项。

## 2. 端到端流程（A1 主链，穿 4 子项目 + 3 契约）

```
[web]  navigator.geolocation 取精确坐标（PII 显式授权）
   │   ——————————————————————————————————— 契约① web→api（写：创建）
   ▼   { 性格文本, 精确坐标(仅透传) }
[api]  录入审核（性格文本 = 注入面首道把关）
   │   ——————————————————————————————————— 契约② api→gen（内部：落地+单次生成）
   ▼   { 性格文本(作数据), 精确坐标 }
[gen]  ① 复核（性格 = 录入审核第二道）
   │   ② 高德逆地理编码(精确坐标)→ 城市；**精确坐标随即丢弃，不落库、不入日志**
   │   ③ home = 城市代表点坐标；1 place = 高德 POI 检索（城市内一个休闲点）
   │   ④ 构建 prompt：**性格作为数据传入，不拼进指令文本**（注入面第二层）
   │   ——————————————————————————————————— 契约③ gen→ai-gateway（内部：LLM 调用）
   ▼   { 系统指令, 性格(数据槽), 现实时间 }
[ai-gateway]  持有 LLM key → 调 DeepSeek → **通用横切 guardrail**（输出审核第三层）
   ▲   返回生成文本
[gen]  组装"诞生"事件 payload（地点=home/place，内容，start，**end**，状态标签）
   ▲   **gen 本期无存储**——只返回 payload（城市/home/place/事件）给 api
[api]  落库：Persona / Place / Event（**存储归属单一 = api**）
   ▲   ——————————————————————————————————— 契约① web←api（读：时间线）
[web]  U1 渲染当天事件序列（列表）
```

> 这条链一次性激活全部 3 条契约与 3 个高影响区，是 A1 的端到端骨架。验收见 [§8](#8-端到端验收a1)。

## 3. 子项目职责（A1 切片）

每个子项目只做 A1 必需的最小切片；专人专项，不跨边界改动。

### 3.1 `apps/web`（前端 · TypeScript/React）

- **U0 表单**：输入性格文本（名字 / 头像本期可选），触发创建。
- **真实浏览器定位**：用 `navigator.geolocation` 取精确坐标，**前置显式授权**（PII，[§5.3](#53-定位-pii收口-web)）。
- **U1 时间线列表**：调 api 读接口，渲染当天事件序列（时间 · 地点 · 在做什么）。
- **契约角色**：api 的消费方；只按 F2 契约调用，不直连 gen / 高德 / LLM。
- **A1 不做**：地图渲染、"此刻"高亮（U2）、历史回看（U4）——见 [§7](#7-首刀明确不做范围外)。

### 3.2 `servers/api`（对外服务 · Node/TypeScript）

- **创建接口**：收性格文本 + 精确坐标，**录入审核（注入面首道）**，转交 gen。
- **存储归属**：本期**唯一持久层**——落库 Persona / Place / Event（[§4](#4-数据模型a1-最小切片)）。
- **读接口**：当天时间线（派生：persona + 当天 → 有序 Event[]）。
- **契约角色**：对 web 提供方；对 gen 消费方（按 gen 既定契约取 payload，不直连 LLM/网关）。
- **A1 不做**：鉴权聚合的完整化、并发幂等（首刀不做并发自链）。

### 3.3 `servers/generation-service`（生成引擎 · Python）

- **无状态计算**：输入 { 性格(数据), 精确坐标 } → 输出 { 城市, home 坐标, 1 place, 诞生事件 payload(含 end) }；**本期不持有任何存储**。
- **高德直连**（高德 key 收口于此，[§5.2](#52-高德-key收口-gen)）：逆地理编码取城市、POI 检索落 1 place。
- **录入审核第二道 + prompt 当数据**：复核性格文本；构建 prompt 时性格走数据槽、不拼进指令（注入面第二层）。
- **经 ai-gateway 调 LLM**：禁止直连 DeepSeek。
- **事件时长**：诞生事件给一个合理 `end`（A1 由生成 + 简单钳制即可，**不引状态机**）。
- **A1 不做**：连贯性状态机、真实情境（天气/POI 营业时间/工作日）、事件自链自调度。

### 3.4 `servers/ai-gateway`（薄 LLM 代理 · Python）

- **薄代理 + 横切**：持有 LLM key、调 DeepSeek、协议适配、超时/重试。
- **输出审核第三层 = 通用横切 guardrail**：对 LLM 输出做注入回显 / 泄漏 prompt / 不安全内容的规则 / denylist 过滤；**不解析业务字段、不读业务语义**——属安全/稳定性横切，与"薄代理"相容（设计理由见 [§6.4](#64-第三层-guardrail-落-ai-gateway-的横切定性)）。
- **契约角色**：gen 的内部提供方，gen 唯一 LLM 入口。
- **A1 不做**：多家 provider / fallback、预算上限与精细限流（永续生成不在首刀，成本压力暂不构成）。

## 4. 数据模型（A1 最小切片）

> 草稿；**最终以 api 机读 schema 为权威（F2）**，feature 只立概念。本期不可变、不清理。

| 实体 | A1 关键字段 | 备注 |
|---|---|---|
| **Persona** | id、owner_user_id、性格文本(用户指定)、城市(逆地理得)、home 坐标 | 用户创建、归用户 |
| **Place** | id、persona_id、类型、坐标(lng,lat) | A1：home（城市代表点）+ 1 个休闲 place（高德 POI） |
| **Event** | id、persona_id、start、**end**、place_id、内容文本、状态标签 | 不可变；A1 即"诞生"事件 |
| Timeline | 派生：persona_id + 日期 → 有序 Event[] | 非持久，按需聚合 |

- **不落精确坐标**：用户定位的精确坐标只在请求链中透传给 gen 做逆地理，**不进任何实体、不入日志**；落库的最细粒度 = 城市 + 城市代表点。
- **A1 不引入**：在途/移动事件、真实情境缓存（天气/POI 营业时间）、跨午夜延续引用。

## 5. 高影响区落点（3/5 ⚡ + guardrails 三层）

A1 激活 [`high-impact.md`](../../rules/high-impact.md) 标 ⚡ 的三类（首刀即生效）。另两类（LLM 成本/限流全量、并发数据一致性）因对应能力被排除（[§7](#7-首刀明确不做范围外)），首刀不构成，留后续刀。

### 5.1 性格 → prompt 注入面（最小必要三层）

用户性格文本被当数据进 LLM = 提示注入面。三层把关，**按子项目收口**：

| 层 | 落点 | A1 做法 |
|---|---|---|
| ① 录入审核（首道） | `api` | 创建入口对性格文本做结构/长度校验 + 最小注入特征 denylist（如"忽略以上指令"类模式）；命中即拒，不进下游 |
| ① 录入审核（复核） | `gen` | 生成前对性格文本二道复核（纵深防御，防 api 漏网） |
| ② prompt 当数据 | `gen` | 性格作为**数据槽**传入，**不拼进指令文本**；指令与用户数据物理分离 |
| ③ 输出审核 | `ai-gateway` | 对 LLM 输出做**通用横切 guardrail**（注入回显/泄漏 prompt/不安全内容过滤），不解析业务字段 |

> "最小必要" = A1 用规则/denylist 级，不引注入检测模型 / PII 脱敏 / 对抗测试（那些是后续加深，见 requirement §15）。

### 5.2 高德 key（收口 gen）

- 高德 key **只在 `generation-service`**；逆地理编码、POI 检索均在 gen 内发起。
- **web 不持高德 key**：故逆地理在 gen 而非 web 做（设计理由见 [§6.3](#63-逆地理在-gen-而非-web)）。
- key **不得硬编码 / 提交**（[`forbidden.md`](../../rules/forbidden.md)）；经环境注入。

### 5.3 定位 PII（收口 web）

- **显式授权**：web 取定位前须用户明确授权。
- **最小化到城市**：精确坐标的唯一用途 = 逆地理取城市；**取得城市后即丢弃精确坐标**，不落库、不入日志。
- 落库最细 = 城市 + 城市代表点；persona 的 `home` **不等于**用户真实住址（= 城市代表点，[§6.2](#62-home--城市代表点的-pii-取舍)）。

### LLM key（收口 ai-gateway）

- LLM provider key **只在 `ai-gateway`**；gen 只持 gateway 凭证，禁止直连 DeepSeek。同样不得硬编码 / 提交。

## 6. 关键实现决策与设计理由（why）

> 以下 6 条是"设定已定稿、但实现有选择/缺口"处，由 feature 牵头人在 brainstorming 质询中定。

### 6.1 单次生成 = 创建时同步生成"诞生"事件（一次）

- **定**：U0 创建 persona 时，**同步**生成首个"诞生"事件，**不另设手动触发接口**；A1 的"单次生成" = 仅此一件。
- **why**：最贴合 requirement §7"创建即从此刻起链、首个为诞生事件"，端到端最短，且与"永续自调度排除"自洽——一刀只证"能生成一个事件"，不证"会持续生成"。

### 6.2 home = 城市代表点的 PII 取舍

- **定**：`home` 落**城市级代表点坐标**；1 个 `place` 经**高德 POI 检索**取城市内真实休闲点。
- **why**：定位 PII 要求"最小化到城市、不存精确坐标"。home 用城市代表点，彻底不依赖、不存储用户真实位置；place 用真实 POI 保留"落在真实城市地图"的产品内核。代价 = A1 即引一次高德 POI 调用（可接受，本就要打通高德 key 收口）。

### 6.3 逆地理在 gen 而非 web

- **定**：浏览器取精确坐标后，经 api 透传给 **gen** 做高德逆地理编码取城市。
- **why**：高德 key 收口 gen（[§5.2](#52-高德-key收口-gen)）。若让 web 直接调高德逆地理，须给前端发高德 key，破坏 key 收口。代价 = 精确坐标须在请求链透传一程；以"只透传、即丢弃、不落库、不入日志"约束兜住 PII。

### 6.4 第三层 guardrail 落 ai-gateway 的横切定性

- **定**：输出审核落 ai-gateway，做法 = **通用横切 guardrail**（注入回显/泄漏 prompt/不安全内容的规则/denylist），**不解析业务字段、不读业务语义**。
- **why**：requirement §11 收口表把"输出审核"放 ai-gateway，但 ai-gateway CLAUDE.md 硬约束"薄代理、不承载业务"——这是跨文档张力（根 §6 牵头人裁）。裁定：把输出审核限定为**安全/稳定性横切**（与"凭证、协议、限流"同类），即与"薄代理"相容，**§11 收口表不变**。边界硬线：一旦需要理解业务语义（如判断"事件是否连贯"），即不属此层、不进 ai-gateway。

### 6.5 api↔gen 同步、Event 落库归属 api

- **定**：api 调 gen，**gen 同步返回 payload（城市/home/place/诞生事件），由 api 落库**；gen 本期无存储。
- **why**：并发幂等被排除（[§7](#7-首刀明确不做范围外)），可走最简同步。存储归属单一（api）→ 契约最清晰、无共享库耦合、无"谁拥有 Event"的并发争用。代价 = 偏离 requirement §9"gen 写 Event"的最终形态；首刀刻意不引共享存储，留后续刀按自链需要再演化（记一笔设计偏离，回流走查 gaps）。

### 6.6 时间线列表承载单条事件

- **定**：U1 是**列表**视图；A1 因不做自链，列表通常只含诞生事件一条。
- **why**：列表是机制（适配 N 条），不为 A1 的 1 条特化。后续刀接入自链后同一视图自然承载多条，无需返工。

## 7. 首刀明确不做（范围外）

A1 **刻意排除**以下项，spec 全文不依赖它们；列出以防范围蔓延：

| 排除项 | 归属后续刀 / 出处 |
|---|---|
| **永续自调度**（事件自链到点触发下一件） | A1 只做"创建时单次生成" |
| **连贯性状态机**（状态转移矩阵、合法转移判定） | requirement §7；A1 单事件无需 |
| **地图渲染**（高德地图 SDK、轨迹连线、U3） | requirement U3 |
| **真实情境**（天气 / POI 营业时间 / 工作日叠加） | requirement §7；A1 不调营业时间，POI 仅取坐标 |
| **并发幂等**（同 persona 并发生成的加锁/幂等） | 高影响"数据一致性"全量；A1 同步单次不构成 |
| **历史回看 3 天**（U4） | requirement U4 |
| 在途/移动事件、跨午夜延续、"此刻"高亮（U2） | requirement U1–U4 / 业务规则 |
| 多家 LLM/fallback、预算上限/精细限流、注入检测模型 | requirement §15 伏笔 |

> 排除 ≠ 否定：均为后续刀或伏笔，A1 只立**能穿透、可验收**的最小骨架。

## 8. 端到端验收（A1）

A1 算跑通 = 以下全部可演示（端到端覆盖，无遗留 TODO）：

1. **U0 穿透**：用户授权定位 → 创建 persona（指定性格）→ 落地城市 + home（城市代表点）+ 1 place（高德 POI）。
2. **单次生成**：创建即同步产出一个"诞生"事件，**含合理 `end` 时间**，经 ai-gateway 调 LLM 得到。
3. **U1**：web 能看到当天事件序列（含该诞生事件）。
4. **3 条契约激活**：web↔api / api↔gen / gen↔ai-gateway 各跑通一次真实调用（schema 与契约测试是 F2）。
5. **高影响 3/5 可验**：
   - 注入面三层就位——性格走录入审核（api 首道 + gen 复核）、prompt 当数据（gen 不拼指令）、输出横切 guardrail（ai-gateway）；
   - 高德 key 仅在 gen、LLM key 仅在 ai-gateway，均不硬编码/提交；
   - 定位仅城市粒度落库，精确坐标不落库、不入日志。

> 端到端验收的可执行细化（步骤 / 断言）放 [`acceptance.md`](acceptance.md)（本期尚未建，随 F2/拆分落地）。
