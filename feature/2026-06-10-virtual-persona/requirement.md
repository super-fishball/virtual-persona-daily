## 1. 产品背景与目标

- **背景**：拟人化陪伴 / 内容消费产品。**核心差异 = 用户造的虚拟人在你所在的真实城市生活**：用户创建虚拟人并指定性格，系统按**当前定位城市**让 ta 落地，ta 的一天在这座城市的真实地图上展开——去真实地点、按真实地理移动、受**真实情境**（天气 / 营业时间 / 工作日）影响。**ta 不上班——替你过你没空过的闲适生活**。
- **要解决的问题**：静态人设没有"活着"的感觉；架空场景缺真实感；纯对话生成无时间锚点与空间锚点。
- **目标**：让每个虚拟人拥有一条**随真实时间推进、落在真实城市地图上、自动生成、连贯可信**的"一天"。
- **产品形态**：以**城市地图为主入口**——首页就是地图，虚拟人光点在真实地图上活动；点进去看个人时间线 / 此刻在做什么 / 当天移动轨迹。
- **价值**：① 真实感（地点 + 时间 + 情境三重锚定）；② 内容自生长；③ 为后续互动提供"当下情境（含真实位置）"上下文。

> **本期已纳入的增强**（来自构思）：1 真实情境注入、2 通勤/移动事件、4 地图为主入口。
> **记入伏笔/未来**：3 成本分层生成、5 同城涌现式社交、6 富事件、7 关注订阅+轻互动、8 跨天轻记忆。

## 2. 术语表

| 术语 | 定义 |
|---|---|
| 虚拟人 Persona | 有固定人设（性格 / 背景 / 说话风格）的虚构角色，绑定一座**设定城市**与一个**家**坐标 |
| 设定城市 | 运营为虚拟人指定的一座**真实城市**，ta 的一天都在这座城市的真实地图上展开 |
| 家 Home | 虚拟人在设定城市里的一个真实坐标点，作息锚点（早晚多在此） |
| 设定地点 / Place | 虚拟人活动的真实地点（家 / 公司 / 常去的店…），各带真实经纬度 |
| 事件 Event | 时间线上的原子节点：某虚拟人在某现实时间、某地点（或在途）、做某事 |
| 移动 / 在途事件 | 一种特殊事件：虚拟人正从 A 前往 B，按真实路网估时；地图上表现为移动中 |
| 时间线 Timeline | 某虚拟人某一天的有序事件序列（派生视图） |
| 现实时间锚点 | 事件对应的真实世界时刻；按设定城市时区推进 |
| 真实情境 | 生成时叠加的真实信号：天气、POI 营业时间、工作日 / 周末 / 节假日 |
| 事件自链 | 每个事件带结束时间，到点自动触发生成下一件，永续推进 |
| AI Gateway（自建） | generation-service 与 LLM 之间的**自建网关**：集中 LLM 密钥、限流 / 重试 / fallback、缓存、成本观测、可选 guardrails |
| 地图锚点 | 事件地点对应的经纬度，用于地图展示与轨迹连线 |
| 状态 | 连贯性判定用的标签（在家 / 外出（在途）/ 用餐 / 休闲 / 睡眠…） |

| 角色 | 能做什么 | 边界（本期） |
|---|---|---|
| 登录用户（toC） | **创建自己的虚拟人（指定性格）**；看自己虚拟人的时间线 / 此刻 / 地图轨迹 | 仅登录；本期**只看自己创建的**（看同城他人 = 多用户社交 → 范围外） |
| 系统（api / generation-service） | 在城市内落地**家（唯一锚点）+ 休闲地点（渐增，真实 POI）**；按现实时间自动生成、校验、持久化 | 用户无感 |
| 运营 / 管理员 | **本期不作 persona 定义者**；仅内容安全审核（见 §11） | 后台界面范围外（见 §13） |

## 4. 核心机制

系统按**事件自链**推进：虚拟人创建后启动，每个事件带结束时间，到点自动生成下一件，永续串成时间线；每个事件落在设定城市的真实地图上。

**4 条核心约束（可判定）：**

| # | 约束 | 怎么判定 |
|---|---|---|
| 1 | 符合现实时间 | 事件状态匹配当前时段；生成时叠加**真实情境**（天气 / POI 营业时间 / 工作日）。下雨在家、店关门就不去。 |
| 2 | 符合地点 / 移动 | 地点取自 persona 的设定地点；相邻地点转移须**可达**（按真实路网估时）——不可达的过渡用**在途事件**表示。 |
| 3 | 符合性格 | 内容 / 说话风格与 persona 性格一致。 |
| 4 | 当天连贯 | 与当天此前事件不矛盾，用**状态机**判（在家→外出→在咖啡馆…合法转移；不重复"第一次起床"）。 |

> 状态机全部状态、真实情境的取数细节留到 §7 业务规则；本节只立机制。

## 5. 关键设定与决策

> 均「可调」。

| # | 议题 | 设定 | 理由 / 伏笔 |
|---|---|---|---|
| A | 性格谁定义 | **用户创建虚拟人并指定性格** | 用户文本进 LLM prompt = **本期高影响**（注入防护 / 内容审核）。 |
| B | 地点 | **用户当前定位城市**（高德定位 → 城市），虚拟人活在这座真实城市 | 动用户真实位置（城市粒度）= **本期高影响**（定位授权、最小化到城市、不存精确坐标）。 |
| C | 生成触发 / 成本 / 失败 / 限流 | **纯事件自链**：每个事件带结束时间，到点生成下一件，对所有虚拟人**永续运行**（不暂停、不分活跃冷）。成本靠事件真实时长**天然有界**（不显式设上限）；**失败 / 限流 / 重试 / 预算由 `ai-gateway` 统一兜**（generation-service 侧：仍失败则留"信息缺失"占位）；天气 / POI 缓存在 generation-service（高德不经 gateway）。 | 永续生成贴合"ta 一直在生活"。**伏笔**：规模大时的成本优化（真没人看就停、成本分层）见 §15。 |
| D | 地图与外部数据源 | **高德**一家覆盖：地图渲染 / POI（营业时间）/ 路径规划（路网估时）/ 定位（当前城市）/ 天气；key 配额计费计入高影响 + forbidden（密钥不得硬编码 / 提交） | 一家收敛依赖。**限制**：高德主要覆盖中国大陆 → 设定城市限国内。备选"自绘虚构地图"见 §15。 |
| E | LLM 接入 | 经**自建 `servers/ai-gateway`** 调 LLM（不直连 provider）：**本期单家 provider、薄代理**；集中 LLM 密钥、限流 / 重试、缓存、成本观测 + **最小必要 guardrails** | 自建子项目；LLM 高影响收口于此。多家 + fallback、guardrails 加深见 §15 |

## 7. 业务规则

- **生成规则（事件自链）**：生成"下一件事"时，喂给 LLM 的输入 = persona 性格 + 当前现实时间 + **真实情境**（天气 / POI 营业时间 / 工作日）+ **当天此前事件**；产出一个事件（地点或在途、内容、**结束时间**）。
- **链起点**：persona 创建时从**此刻**起链，首个为"诞生"起点事件（如"刚来到这座城市"）；**不补创建前的时段**。
- **事件时长**：LLM 产出 + 规则**合理性钳制**（落该类型合理区间，如睡眠 7–9h、用餐 0.5–1.5h）；通勤 / 在途时长由高德路网估时覆盖。
- **连贯性（状态机）**：状态 = 在家 / 外出（在途）/ 用餐 / 休闲 / 睡眠…；相邻状态须**合法转移**；地点须**可达**（不可达就插一个**在途事件**过渡，按高德路网估时）。
- **跨午夜 / "当天"定义**：当天 = **自然日**窗口；跨午夜事件**不切**（一条 start/end 跨两天），**归属开始日**、在所跨两天都展示；生成今天首件时**引用这条延续事件**作上下文（否则不知 ta 在睡）。
- **现实时间锚定**：按**设定城市时区**（= 用户定位城市的时区）。
- **失败 / 限流**：见 §5 的 C（退避重试、留占位、缓存天气 / POI）。

> 状态机只列状态集 + 判定原则，完整转移矩阵属实现细节，spec 阶段再细化。本期**不做数据清理**（旧数据只是不展示）。

## 8. 数据模型

> 草稿；最终以 api 机读 schema 为权威，feature 只放指针。

| 实体 | 关键字段（草稿） | 备注 |
|---|---|---|
| 用户 User | id、登录标识 | 拥有自己的 Persona |
| 虚拟人 Persona | id、owner_user_id、name、**性格（用户指定）**、城市（来自定位）、家坐标、头像? | 用户创建、归用户 |
| 设定地点 Place | id、persona_id、名称、类型（家 / 咖啡馆 / 公园 / 书店 / 餐饮…）、坐标(lng, lat) | 家=唯一锚点；休闲地点首次用到时落 POI、缓存渐增 |
| 事件 Event | id、persona_id、**start、end**、place_id 或 在途(from→to)、内容文本、状态标签、生成上下文摘要、占位标记(失败时) | **不可变**；本期不清理 |
| 时间线 Timeline | 派生：persona_id + 日期 → 有序 Event[] | 非持久，按需聚合；只查最近 3 天 |

> 天气 / POI 营业时间是**缓存的外部数据**（按城市 / POI 缓存），不进核心实体。

## 9. 跨子项目分工与数据流

> 本需求采用"对外服务 vs 内容生成"切法，**与 template 的 web/api/worker 不同**（结构按产品需要定）。

| 子项目 | 职责 |
|---|---|
| `apps/web`（前端） | 创建虚拟人（U0）、**地图为主入口**（U3）、时间线 / 此刻（U1/U2）、历史 3 天（U4）；高德地图 SDK 渲染 + 定位取城市；契约**消费方** |
| `servers/api`（对外服务） | Persona / Place / Event 读写接口（时间线 / 当前 / 地图轨迹）、鉴权；**只服务不生成**；对 web 的契约**提供方** |
| `servers/generation-service`（生成引擎） | 高德（POI / 路网 / 天气）+ 连贯性状态机 + 内容审核 + **事件自链自调度**；**经 ai-gateway 调 LLM**；写 Event |
| `servers/ai-gateway`（自建网关） | 接入 LLM provider（**本期单家、薄代理**）、**集中 LLM 密钥**、限流 / 重试、缓存、成本观测 / 预算、**最小必要 guardrails**；对 generation-service 的内部契约**提供方** |

**数据流：**

```
用户创建 persona (web→api) → 启动链
  → generation-service 按"上一事件 end"自调度
  → 取 persona + 当天事件 + 真实情境(高德直连 / 天气缓存)
  → 经 ai-gateway 调 LLM 生成下一事件(含 end)
  → 连贯校验 + 内容审核 → 写 Event
  → api 读取 → web 展示(时间线 / 地图)
```

> 契约边界：**web↔api（对外）、api↔generation-service（内部）、generation-service↔ai-gateway（内部）**——3 条契约，dogfood 料更足。
> 此切法把原 **G3**（worker/api 谁生成）的模糊**自然消解**；但 workflow 缺"通用拆分判据"这个缺口仍在（见 [gaps.md](docs/walkthrough/2026-06-08-virtual-persona-daily/gaps.md) G3）。

## 10. 接口概览

> 只列意图与方向；权威 = api 机读 schema，feature 只放指针。

| 方向 | 接口意图 |
|---|---|
| web → api（写） | 创建虚拟人（性格 + 城市）→ 建 Persona + 启动事件链 |
| web → api（读） | 我的虚拟人列表 / 详情；当天时间线；当前事件（"此刻"）；地图轨迹（当天事件 + 坐标）；历史（过去 3 天） |
| 内部 | generation-service 写 Event（共享存储或内部接口）；**generation-service → ai-gateway 调 LLM**；高德由 generation-service 直连（均不暴露给 web） |

- **契约提供方 = api**（对 web）；内部边界：**api ↔ generation-service**、**generation-service ↔ ai-gateway**。
- 高德直连封在 generation-service、LLM 经 ai-gateway；均不进对 web 的契约。

## 11. 高影响区与风险

| 区域 | 风险 | 归类 |
|---|---|---|
| 用户性格 → LLM prompt | 用户文本进 prompt = **注入面**；**最小必要三层**：录入审核（api/gen）+ prompt 当数据（gen）+ 输出审核 / 注入检测（ai-gateway） | 高影响（安全） |
| 用户定位 | 城市粒度真实位置 = **PII**；定位授权、最小化到城市、不存精确坐标 | 高影响（隐私） |
| 密钥 | **LLM provider key 集中在 `ai-gateway`**，generation-service 只持 gateway 凭证；**高德 key 在 generation-service**；均**不得硬编码 / 提交** | 高影响 + forbidden（密钥） |
| LLM 成本 / 限流 / 失败 | **由 `ai-gateway` 统一兜**（预算上限、限流、重试 / fallback、缓存、观测） | 高影响（集中管控） |
| 数据一致性 | 事件自链**并发生成同一 persona** 需幂等 / 加锁；Event 不可变 | 高影响 |

> 高影响**按子项目收口**：LLM 接入相关 → `ai-gateway`；生成逻辑 / 注入审核 → `generation-service`；定位 / PII → `web`。

## 12. 验收标准

- **端到端**：用户创建 persona → 事件自链随时间推进产出**连贯**的当天时间线 → web 能看 时间线 / 此刻 / 地图轨迹 / 历史 3 天。
- **4 约束**：分别可验收——时段相容（含真实情境：下雨在家、店关门不去）、地点可达 / 在途过渡、性格一致、当天不矛盾。
- **失败注入**：LLM 失败时 ai-gateway 重试 / fallback；仍失败留"信息缺失"占位、链下次接续。
- **成本 / 限流**：ai-gateway 预算上限与限流生效。
- **安全 / 隐私**：用户性格内容审核 + 注入防护生效；定位仅城市粒度、不存精确坐标。
- **地图**：轨迹点与事件一致、与时间线联动。

## 15. 开放问题 / 伏笔

### 伏笔（仅记录，将来可能做）

- 成本优化：真没人看就停、成本分层。
- 富事件（心情 / 动态 / 配图）、关注订阅 + 轻互动、跨天轻记忆。
- 历史回看 > 3 天；数据清理 / 归档（量大再做）。
- 自绘风格化虚构地图（轻量 / 出海版备选；本期已定用高德真实地图）。
