## 1. 背景 / 需求来源

A1 是整套产品的第一刀，用最小切片一刀穿透 4 子项目 + 3 契约 + 3 高影响区（总 spec §1）。本子项目 spec 只覆盖 gen 在主链中的那一段：

> `[api] —契约②→ [gen] ①复核 ②逆地理 ③home/place ④prompt当数据 —契约③→ [ai-gateway] ⑤LLM ⑥组装payload→ [api]`（总 spec §2 主链的 `[gen]` 段）

gen 的契约角色（子项目 CLAUDE.md §1 / §7）：
- 对 `servers/api`：**契约② 提供方**——产出 payload 供后端聚合落库。
- 对 `servers/ai-gateway`：**契约③ 消费方**——所有 LLM 调用一律经网关，**禁止直连 DeepSeek**（根 forbidden + 本地 forbidden）。

关联术语见 [`../../../context/domain-glossary.md`](../../../context/domain-glossary.md)。

---

## 2. 目标

实现无状态的 `POST /generate`（契约②）：一次调用完成「落地计算 + 单次生成诞生事件」，返回结构化 payload（**无存储、不铸 DB id**，由 api 落库）。同时就位 gen 承担的两道高影响：**录入审核第二道**（注入面纵深防御）、**prompt 当数据**（指令/用户数据物理分离），并守住**精确坐标用后即丢**与**高德 key 收口**。

---

## 3. 范围

### 3.1 做（A1 gen 切片）

- `POST /generate`：契约② 的提供方实现，6 步 pipeline（§5）。
- 高德直连：逆地理编码取城市、行政区划取城市代表点、POI 检索落 1 place（高德 key 收口于 gen，§7.3）。
- 录入审核第二道（结构化规则启发式，§9·决策③）。
- prompt 当数据构建（性格走数据槽、不拼指令；§6 / §9·决策②）。
- 经 ai-gateway（契约③）调 LLM；按 §6 接住 400/422/500/502 + 超时。
- 组装 birthEvent payload（start/end/placeRef/content/statusTag 全由 gen 机械计算，LLM 只产 content；§9·决策⑤）。

### 3.2 不做（范围外，沿总 spec §7，gen 视角）

- **无任何存储 / 不铸 id**（总 spec §3.3 / §6.5）：只返回 payload。
- **不引状态机 / 状态转移**（end 仅简单钳制）。
- **不做永续自调度 / 事件自链**（A1 只此一次同步生成）。
- **不做并发幂等**（同步单次；G-1 已知：超时重试非幂等会重复烧高德/LLM 配额，故本期**不重试**，§6）。
- **不引注入检测模型 / PII 脱敏模型 / 对抗测试**（录入审核第二道止于规则/启发式，§9·决策③）。
- **不做真实情境**（天气 / POI 营业时间 / 工作日）：POI 仅取坐标，不查营业时间。
- 不绕过网关直连 LLM；不单方改动既有契约②③（越界即停，走 feature 层——本地 scope.md）。

## 4. 接口 / 数据结构（提供方实现，权威以契约② schema 为准）

`POST /generate`，请求/响应结构**以 [`../contracts/api-gen.openapi.yaml`](../contracts/api-gen.openapi.yaml) 为唯一权威**，此处只记实现要点：

| 项 | 要点 |
|---|---|
| 请求 `GenerateRequest` | `personality`(string 1–500)、`location`(Coordinate, **PII/瞬态**) |
| 响应 `GenerateResponse` | `city`、`home`(Coordinate)、`place`(PlacePayload)、`birthEvent`(EventPayload) |
| `EventPayload` | `start`/`end`(date-time)、`placeRef`(enum `home`/`place`)、`content`、`statusTag` |
| 对外错误码 | **仅 `400` / `502`**（契约② 只声明这两类非 2xx）——映射见 §6 / §9·决策① |

实现须配**契约测试**（提供方侧验证实现符合契约② schema；防漂移，split §1）。

## 5. 内部 6 步 pipeline

`POST /generate` 内部按序执行；任一步失败按 §6 错误处理：

| 步 | 动作 | 高影响 / 约束 |
|---|---|---|
| **① 性格复核** | 录入审核**第二道**：对 `personality` 做结构化规则启发式校验，命中即拒（→ 400 `personality_rejected`）。 | 注入面第二层（纵深，api 首道在前）。做法见 §9·决策③ |
| **② 逆地理 → 城市** | 高德逆地理编码（精确 `location`）→ 设定城市；**精确坐标随即丢弃**——不落库、不入日志、不进任何后续步骤的入参（除② 自身）。 | 定位 PII 收口（§7.2）。参数见 §9·决策④ |
| **③ home + place** | `home` = **城市代表点**坐标（行政区划取，不依赖用户精确位置）；`place` = 高德 POI 检索（**围绕城市代表点**）城市内 1 个休闲点。 | G-2（§7.1）；PII：POI 不用精确坐标（§9·决策④） |
| **④ 构建 prompt** | 构建 `systemInstruction`（静态任务模板 + city/place 上下文）；`personality` **只**走契约③ 数据槽字段、**不拼进指令**；`realTime` 走契约③ 专属字段。 | 注入面第二层「prompt 当数据」（§6 / §9·决策②） |
| **⑤ 经 ai-gateway 调 LLM** | 调契约③ `POST /v1/complete`；接住 400/422/500/502 + 超时（§6）。 | 禁直连 DeepSeek；超时>30s 不重试 |
| **⑥ 组装 payload** | LLM 仅产 `content` 文本；`start`/`end`/`statusTag`/`city`/`home`/`place`/`placeRef` 全由 gen **机械计算**。返回 `GenerateResponse`。 | §9·决策⑤（结构不靠 LLM） |

> **placeRef = `home`（G-2）**：A1 诞生事件落 `home`；§3 检索的休闲 `place` 本期**暂不被事件引用**，**仍保留**——穿透高德 POI 契约路径（dogfood）+ 后续刀即用免返工（split §4·G-2）。

## 6. 消费契约③ 的对接（v0.1.1-a1）

### 6.1 systemInstruction / personality / realTime 分离（「性格当数据」落法）

契约③ 取 **3 个分立字段** `systemInstruction` / `personality` / `realTime`，保持「指令 vs 用户数据」物理分离（总 spec §6.4·GW-2）。gen 侧落法：

- `systemInstruction`：静态任务模板 + **city / place 上下文**。**已裁决：city/place 进 systemInstruction（理由：契约③ 无对应字段；city 是逆地理派生的系统事实、place type 来自 POI，均非用户自由文本，不构成注入向量——"指令不含用户数据"防的是用户可控文本注入指令，不是系统派生事实）。**
- `realTime`：**已裁决：走契约③ 专属 `realTime` 字段，不塞进 systemInstruction（理由：契约③ 已有独立字段，应各归其位；§9·决策② 调整项）。**
- `personality`：**只**置入契约③ `personality` 数据槽，**永不** format/拼接进 `systemInstruction`。实现须加**断言 + 单测**守住此分离（性格字符串不得出现在 systemInstruction 构造中）。

### 6.2 超时与重试（G-1 非幂等）

契约③ 描述：网关最坏时延 ≈30s（单次上游超时 15s + 最多 1 次重试 0.5s 退避）。gen 据此：

- gen 调用契约③ 的**自身超时 > 30s**（实现取一个 >30s 的具体阈值，留 plan 定，建议 ~35s）。
- **不再叠加 gen 侧重试**——G-1 已知：A1 重试非幂等，会重复消耗高德/LLM 配额（split §4·G-1 / 总 spec §10）。

### 6.3 错误码映射（契约③ 内部码 → 契约② 对外码）

**已裁决**（§9·决策①，牵头人采纳）：

| 来源 | gen 对外 HTTP | gen 的 `Error.code` | 理由 |
|---|---|---|---|
| ① 性格复核拒绝 | **400** | `personality_rejected` | gen 自身入口判定 |
| 请求结构非法（不符契约②） | **400** | `invalid_request` | 契约② `400` 语义 |
| 契约③ 返回 **422** guardrail_blocked | **502** | `generation_blocked` | **已裁决：独立 code（理由：api/web 要分清"被 guardrail 挡=内容问题、不可重试" vs "下游真挂=或可稍后重试"；独立 code 近零成本、信号更准）**。block-only → **不重试** |
| 契约③ 返回 **400** invalid_request | **502** | `upstream_unavailable` | 这是 **gen 自身构造请求的 bug**；对外 502 **+ 内部告警（不可静默）** |
| 契约③ 返回 **500** internal_error | **502** | `upstream_unavailable` | 网关内部失败 |
| 契约③ 返回 **502** / 超时(>30s) | **502** | `upstream_unavailable` | DeepSeek 不可用；**不叠加重试**（G-1） |
| 高德逆地理 / POI 失败 | **502** | `upstream_unavailable` | 下游不可用 |

> 所有非 2xx 用契约② `Error{code,message}`；`message` **不得回显** prompt / personality / 精确坐标（forbidden + PII）。

## 7. 高影响区落点

### 7.1 G-2：placeRef = home（休闲 place 暂不被引用但保留）

诞生事件 `placeRef=home`；§3 检索的 1 个休闲 place **保留**于响应 `place`，本期事件不引用（split §4·G-2，已裁决保留）。

### 7.2 定位 PII：精确坐标用后即丢（收口链中"取数"一手）

- 精确 `location` 唯一用途 = ② 步逆地理取城市；**取得城市后即丢弃**。
- **不落库**（gen 本无存储）、**不入日志**（access/error/trace 均不记 `location`）、**不进 POI 检索**（POI 围绕城市代表点，§9·决策④）。
- 落库最细 = 城市 + 城市代表点（home ≠ 用户真实住址，总 spec §6.2）。
- forbidden：禁止日志输出含隐私的完整 prompt / 内容。

### 7.3 高德 key：收口 gen

- 高德 key **只在 gen**；逆地理、行政区划、POI 检索均在 gen 内发起。
- key **不得硬编码 / 提交**（forbidden），经环境注入。

### 7.4 LLM 调用：经网关、有超时

- 禁止直连 DeepSeek；唯一 LLM 入口 = 契约③（本地 forbidden）。
- LLM 调用有超时（§6.2）；A1 不重试（G-1）。

## 8. 验收标准（gen 切片）

gen 算跑通 = 以下全部可验（对齐总 spec §8，gen 视角）：

1. **契约② 激活**：`POST /generate` 真实跑通一次，响应符合契约② schema（契约测试绿）。
2. **6 步 pipeline 端到端**：输入 {personality, location} → 输出 {city, home, place, birthEvent(含合理 end)}。
3. **契约③ 消费**：经 ai-gateway 真实调一次 LLM 得 content；400/422/500/502 + 超时各有对应处理（§6.3，单测覆盖映射表每一行）。
4. **注入面第二层可验**：① 录入审核第二道命中样例即拒（400 `personality_rejected`）；④ systemInstruction 构造中**断言**不含 personality（单测）。
5. **PII 可验**：精确坐标不落库（gen 无存储）、不入日志（日志断言无 `location`）、POI 检索入参 = 城市代表点而非精确坐标（单测）。
6. **key 收口可验**：高德 key 仅在 gen、不硬编码（扫描）；无直连 DeepSeek 旁路（代码审查 + 仅经契约③）。

> 端到端可执行细化（步骤/断言）归 feature `acceptance.md`（待建，随拆分落地）；本 spec 验收为 gen 单元 + 契约测试层。

## 9. 关键实现决策清单（已裁决 · 牵头人评审采纳）

> 原为 brainstorming 的 5 处实现歧义，经牵头人评审定稿；**不再留"待裁决"**。3 处对原 lean 的调整以 **【调整】** 标注，其余按原倾向采纳。

### 决策① 错误码映射 —— 422 用独立 code `generation_blocked` 挂 502
- **已裁决**：映射表见 §6.3。422 guardrail_blocked → 502 但用**独立 code** `generation_blocked`（非 `upstream_unavailable`）。
- **理由**：api/web 要分清"内容被挡（不可重试）" vs "下游真挂（或可重试）"；独立 code 近零成本、信号更准。gen 自造的 400/500 → 502 + 内部告警（不静默）。

### 决策② systemInstruction 构造 —— 【调整】realTime 走专属字段
- **已裁决**：`realTime` 走契约③ **专属 `realTime` 字段**，**不**塞进 systemInstruction；仅 city/place（契约③ 无对应字段）进 systemInstruction。personality 只走数据槽、永不进指令（加断言+单测）。
- **理由**：契约③ 已有独立 realTime 字段，各归其位；city 为系统派生事实、非用户自由文本，进指令不违"指令不含用户数据"的本意（该规矩防用户可控文本注入，非系统事实）。

### 决策③ 录入审核第二道 —— 结构化规则启发式（lean b）
- **已裁决**：第二道 = **结构/启发式规则**（注入标记如 `system:`/`assistant:`/`###`/试图闭合上下文、指令密度、超长 token blob/base64、charset 异常），命中即拒。**非**纯 denylist、**非** ML 打分。
- **理由**："语义级 but A1 不引检测模型" 落在结构化启发式；这是纵深第二道（api 首道在前），**不必穷尽**，cheap、命中即拒即可。

### 决策④ 高德参数 —— 【调整】POI 围绕城市代表点（隐私对齐）
- **已裁决**：
  - 逆地理：`/v3/geocode/regeo`，city 提取带**直辖市回退**（city 空 → district/province）。
  - 城市代表点：行政区划 `/v3/config/district` 取该城 adcode 中心点（regeo 不直接给城市中心）。
  - POI：`/v3/place/around` **围绕城市代表点**检索（**绝不用精确坐标**），radius≈5000，types = **固定小集**（咖啡馆 / 公园 / 书店，对齐契约 `PlacePayload.type` 示例），**取首条**（确定性、利于契约测试）。
- **理由**：POI 用城市代表点与"精确坐标逆地理后即丢"对齐，防精确坐标泄漏到 POI 这步；取首条确定性便于测试。

### 决策⑤ start/end/statusTag —— LLM 只产 content，结构全由 gen 机械计算（锁死）
- **已裁决**：LLM **仅产 `content` 文本**；`start`/`end`/`statusTag`/`city`/`home`/`place`/`placeRef` 全由 gen 机械计算：
  - `start` = 此刻；时区 A1 **固定 UTC+8**（高德属中国域，作 A1 简化）。
  - `end` = `start + ~60min`，钳制 `[15min, 4h]` 且 **≤ 当日 23:59**（不跨午夜，合总 spec §4）。
    - **午夜边界优先级（已裁决）**：当 `start` 接近午夜（约 >23:44）时，"end ≤ 当日23:59" 与 "end ≥ start+15min 下限" 冲突——**「≤ 当日23:59」优先**（理由："不跨午夜"是总 spec §4 当天时间线的硬约束，高于时长下限）；该罕见场景 `end` 可 `<15min`，**接受**。
  - `statusTag` = **固定常量**（如 `初来乍到`），A1 无状态机。
  - `placeRef` = `home`（G-2）。
- **理由**：契约③ 只回 `text`，gen 控结构 = 不解析 LLM 输出凑字段（脆弱）、LLM 输出面最小（guardrail 只看 text），与"薄 ai-gateway、结构不靠 LLM"的整体架构一致。后续刀再派生时区 / 引状态机。
