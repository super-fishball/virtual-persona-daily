# spec/ — 技术规格说明

> 存放本服务的技术规格说明（做什么、接口 / 数据结构设计）。

## 命名

`YYYY-MM-DD-spec-<标识>.md`，按任务命名。例：`2026-06-04-spec-daily-event.md`

## 必填字段

- **背景 / 需求来源**：关联需求或 Bug；跨子项目时链接 feature 总 spec 与契约。
- **目标**。
- **范围**：涉及模块 / 接口；明确"不做"什么（划边界）。
- **接口 / 数据结构**：对外事件契约定义（作为 `servers/api` 的提供方，须与 feature 层契约一致）；对 `servers/ai-gateway` 的消费契约（按网关既定契约）。
- **生成设计**：prompt 模板要点、输入/输出结构、成本与调用量预期。
- **验收标准**。
- **关联术语**：引用 `context/domain-glossary.md`。
