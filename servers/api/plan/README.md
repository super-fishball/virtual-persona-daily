# plan/ — 实现计划

> 实现计划（怎么做、AI 参与范围、不允许 AI 改动的范围）。

## 命名

`YYYY-MM-DD-plan-<标识>.md`，与对应 spec 同标识。

## 必填字段

- **实现步骤**。
- **AI 参与范围**：允许 AI 起草 / 修改的部分；**禁止 AI 改动**的部分（认证 / 计费 / 迁移等高影响区，对应 development-guard 禁改范围）。
- **测试计划**：先写失败测试（TDD），覆盖契约与验收点；不得为适配实现弱化测试。
- **风险与回滚**：是否触及 high-impact、schema 迁移回滚方案、对前端契约的影响。
