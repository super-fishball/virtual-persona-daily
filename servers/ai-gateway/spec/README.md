# spec/ — 技术规格说明

> 存放本服务的技术规格说明（做什么、接口 / 数据结构设计）。

## 命名

`YYYY-MM-DD-spec-<标识>.md`，按任务命名。例：`2026-06-04-spec-completion-proxy.md`

## 必填字段

- **背景 / 需求来源**：关联需求或 Bug；跨子项目时链接 feature 总 spec 与契约。
- **目标**。
- **范围**：涉及模块 / 接口；明确"不做"什么（划边界）——尤其守住"薄代理"边界，不纳入业务逻辑。
- **接口 / 数据结构**：对内契约定义（作为 `servers/generation-service` 的提供方，须与 feature 层契约一致）；上游 DeepSeek 的协议适配要点。
- **横切策略**：凭证持有方式、超时 / 重试 / 限流 / 熔断、错误透传约定。
- **验收标准**。
- **关联术语**：引用 `context/domain-glossary.md`。
