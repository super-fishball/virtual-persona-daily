# plan/ — 实现计划

> 实现计划（怎么做、AI 参与范围、不允许 AI 改动的范围）。

## 命名

`YYYY-MM-DD-plan-<标识>.md`，与对应 spec 同标识。

## 必填字段

- **实现步骤**。
- **AI 参与范围**：允许 AI 起草 / 修改的部分；**禁止 AI 改动**的部分（对内契约、凭证处理、限流 / 熔断等高影响区，对应 development-guard 禁改范围）。
- **测试计划**：先写失败测试（TDD），覆盖契约与验收点；不得为适配实现弱化测试。
- **风险与回滚**：是否触及 high-impact、对 `servers/generation-service` 契约的影响、凭证安全、超时 / 重试 / 限流是否就位、是否破坏"薄代理"边界。
