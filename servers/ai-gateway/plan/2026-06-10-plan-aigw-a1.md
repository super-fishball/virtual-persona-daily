# ai-gateway · A1 首刀实现计划（POST /v1/complete · 薄代理 + 第三层 guardrail）

> **For agentic workers:** ④ TDD 执行须经牵头人启动；本计划用 checkbox（`- [ ]`）逐步跟踪。**本步只产 plan 文档，代码是 ④。**

**Goal:** 实现契约③ `POST /v1/complete`——三分立字段机械映射 → 调 DeepSeek → 第三层 block-only guardrail → 返回 `{text}`，错误透传 422/502。

**Architecture:** FastAPI 路由薄；职责拆 6 个聚焦文件（models / mapping / deepseek / guardrail / service / 路由+错误处理）。纯函数（mapping、guardrail）与有副作用的 provider 分离，便于单测。guardrail 只对**输出文本**做规则匹配，不读业务语义（守"薄"，GW-1 硬线）。

**Tech Stack:** Python 3.12 · FastAPI · httpx（异步，调 DeepSeek）· pytest（`TestClient` + `httpx.MockTransport`）· ruff · mypy。

> 关联：① spec [`../spec/2026-06-10-spec-aigw-a1.md`](../spec/2026-06-10-spec-aigw-a1.md)、契约③ [`../contracts/gen-aigw.openapi.yaml`](../contracts/gen-aigw.openapi.yaml)。

## 实现层取值（spec §5.2 留给 plan 的值 · 本计划定死）

| 项 | 值 | 理由 |
|---|---|---|
| 每次尝试超时 | `httpx.Timeout(30.0, connect=5.0)` | LLM 补全单次可数秒；connect 短失败快 |
| 最大重试 | `MAX_RETRIES = 1`（总尝试 2 次） | **G-1 非幂等**——重试会重复烧 DeepSeek 配额，刻意取小，只兜瞬时抖动 |
| 重试退避 | `BACKOFF_SECONDS = 0.5`（固定） | 简单、足够隔开瞬时抖动；不做指数退避（YAGNI） |
| 重试触发 | 仅 `httpx.TimeoutException` / `httpx.TransportError` / 上游 5xx | 4xx 不重试（非瞬时），直接 502 |
| 模型 / base_url | `deepseek-chat` / `https://api.deepseek.com`（base_url 可经 `DEEPSEEK_BASE_URL` 覆盖） | 单家 DeepSeek（CLAUDE.md §4） |

`app/deepseek.py`:

```python
import asyncio
import os

import httpx

from app.errors import UpstreamUnavailable

_BASE_URL = os.environ.get("DEEPSEEK_BASE_URL", "https://api.deepseek.com")
_MODEL = "deepseek-chat"
_TIMEOUT = httpx.Timeout(30.0, connect=5.0)
MAX_RETRIES = 1  # 总尝试 = 2；G-1 非幂等，刻意取小
BACKOFF_SECONDS = 0.5


async def call_deepseek(
    messages: list[dict[str, str]],
    *,
    transport: httpx.BaseTransport | None = None,
) -> str:
    api_key = os.environ.get("DEEPSEEK_API_KEY")
    if not api_key:
        # 不泄露细节；仅作不可用透传。
        raise UpstreamUnavailable("missing DEEPSEEK_API_KEY")

    payload = {"model": _MODEL, "messages": messages}
    headers = {"Authorization": f"Bearer {api_key}"}
    last_error = "upstream failed"

    for attempt in range(MAX_RETRIES + 1):
        try:
            async with httpx.AsyncClient(
                base_url=_BASE_URL, timeout=_TIMEOUT, transport=transport
            ) as client:
                resp = await client.post("/chat/completions", json=payload, headers=headers)
            if resp.status_code >= 500:
                last_error = f"upstream {resp.status_code}"  # 可重试
            elif resp.status_code >= 400:
                raise UpstreamUnavailable(f"upstream {resp.status_code}")  # 不重试
            else:
                data = resp.json()
                return str(data["choices"][0]["message"]["content"])
        except (httpx.TimeoutException, httpx.TransportError) as exc:
            last_error = type(exc).__name__  # 不记录 body/prompt
        if attempt < MAX_RETRIES:
            await asyncio.sleep(BACKOFF_SECONDS)

    raise UpstreamUnavailable(last_error)
```

```python
_INJECTION_ECHO = (
    "忽略以上指令",
    "忽略上述指令",
    "ignore the above",
    "ignore previous instructions",
    "disregard previous",
)
# 不安全内容 denylist（起步词表，可扩）。
_UNSAFE_TERMS = ("炸弹制作方法", "制毒")
# 泄漏 prompt：系统指令被原样回显的最小判定长度。
_LEAK_MIN_LEN = 20


def scan_output(text: str, system_instruction: str) -> str | None:
    """对输出文本做三类规则匹配；命中返回原因码，否则 None。

    block-only、只看输出文本、不读业务语义（GW-1 硬线）。
    """
    lowered = text.lower()
    for needle in _INJECTION_ECHO:
        if needle.lower() in lowered:
            return "injection_echo"

    si = system_instruction.strip()
    if len(si) >= _LEAK_MIN_LEN and si in text:
        return "prompt_leak"

    for term in _UNSAFE_TERMS:
        if term in text:
            return "unsafe_content"

    return None
```
