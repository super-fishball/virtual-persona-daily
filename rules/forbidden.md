# 禁止事项（forbidden）

> AI Agent 的绝对红线。命中任意一条 → **立即停止，提示人工处理**，不得自行绕过。
> 本文件为全局基线，子项目只能更严，不得放宽。

## 凭证与机密

- 禁止读取、打印、提交任何密钥 / 凭证 / `.env` / token / 证书私钥。
- 禁止把机密写入代码、日志、注释、测试夹具。

## 数据与隐私

- 禁止在日志、测试夹具、提交内容中使用真实用户数据（PII）。
- 禁止外泄、或将真实生产数据复制到非生产环境。

## 基础设施与发布

- 禁止修改 CI/CD 配置、部署脚本、分支保护规则——除非任务 spec 明确授权。
- 禁止执行任何**生产环境操作**（部署、改生产配置、操作生产数据库）。生产操作一律人工执行。
- 禁止对共享分支 `force-push` 或改写已推送的 git 历史；禁止无迁移脚本 / 回滚方案地 drop 或变更数据库表。

## 测试完整性

- 禁止删除、跳过、或弱化测试**以适配当前实现**（如把断言改成迁就 bug 的值）。
- 测试只能反映**需求/spec 的真实预期**，不得为了"让测试通过"而造假。

## 结果完整性与诚实

- 禁止声称**未经实际验证**的成功——如未运行测试却报告"测试通过"、未构建却称"构建成功"。
- 禁止以掩盖错误/警告的方式伪造通过：大面积 `@ts-ignore` / `eslint-disable`、注释或跳过失败检查、吞掉异常等手段强行让门禁变绿。
- 状态须如实报告：失败就说失败并附输出，跳过就说跳过。

## 任务边界

- 禁止在无 `spec` / `plan` 依据下改动代码（简单需求需开发者明确豁免）。
- 禁止**跨子项目边界**改动（专人专项）：在 `apps/web` 的任务不得直接改 `servers/api`，跨子项目需走 feature 层契约先行再拆分。
- 禁止超出 plan 声明的"AI 参与范围"或触碰 plan 标注的"禁改范围"。

## 依赖与架构

- 禁止擅自引入新依赖、新外部服务、或调整目录结构——这类属高影响，须走升级评审（见 [`review-gates.md`](review-gates.md)）。
