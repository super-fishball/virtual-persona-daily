# 高影响区（high-impact）

> 命中高影响区的改动 → 必须**升级评审**（见 [`review-gates.md`](review-gates.md)），由责任域成员参与确认。
>
> 📌 **路径清单只有一份机读权威源** = [`high-impact.paths`](high-impact.paths)（hooks / CI 硬门禁 / 开发中自检 共用）。本文件只解释**判定标准与 why**，**不复述具体路径**，避免散文 + 配置两份漂移。

## 判定标准

满足以下任一即视为高影响：

- **影响面大**：被多处依赖、改动会波及多个模块或子项目。
- **难回滚**：数据迁移、不可逆的 schema 变更、历史数据改写。
- **涉及钱 / 权限 / 数据安全**：计费、支付、认证授权、用户数据隔离。
- **对外契约**：跨子项目 API、对外开放接口、消息格式。
- **生产稳定性**：生产配置、限流熔断、定时任务、并发与事务边界。

## 通用高影响类目

| 类目 | 说明 |
|---|---|
| 认证 / 授权 | 登录、会话、权限校验、token 签发 |
| 支付 / 计费 | 金额计算、对账、退款 |
| 数据迁移 | schema 变更、批量数据改写 |
| 跨子项目契约 | API 接口、数据结构、消息格式 |
| 生产配置 | 环境变量、部署参数、特性开关 |
