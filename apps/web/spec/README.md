# spec/ — 技术规格说明

> 存放本子项目的技术规格说明（做什么、接口 / 数据结构设计）。

## 命名

`YYYY-MM-DD-spec-<标识>.md`，按任务命名。例：`2026-06-04-spec-login.md`

## 必填字段

- **背景 / 需求来源**：关联需求或 Bug；跨子项目时链接 feature 总 spec。
- **目标**：这个任务要达成什么。
- **范围**：涉及模块 / 页面；明确"不做"什么（划边界）。
- **接口 / 数据结构**：对接的后端契约（引用 feature 层契约）、前端数据结构 / 组件接口。
- **验收标准**：怎样算完成；端到端部分指向 feature 层验收。
- **关联术语**：引用 `context/domain-glossary.md`，避免歧义。
